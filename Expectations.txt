Heres my rules and expectations when developing this plugin

* Use the modern Paper API only.
* Do not use deprecated APIs or legacy Bukkit methods.
* Setup the prefixes and messages to use gradients, chat colour codes and <grey> colours which the chat side is all handled by my own custom plugin
* Every message in the plugin must be configurable.
* Have a configurable prefix in the config.
* onEnable and onDisable must send configurable startup/shutdown messages using the prefix.
* No strings should be hardcoded.
* If tasks need to be run asynchronously, always use async tasks properly.
* Add tab completion for all commands.
* Always include a command to reload the plugin configuration.
* Keep the code clean, modular, and production quality.
* Automatically update the build up 0.1 
* After each update put the summary in devlog.md and label per update not by date
* Suggest any new features in potential-roadmap.md
